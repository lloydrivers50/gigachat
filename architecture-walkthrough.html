gigachat · system design · the target architecture

One travel request, walked through the whole system

A user types "book me a train to Manchester tomorrow" into a corporate-travel chat assistant. This page traces that one request through gigachat's target async architecture — and at every hop, asks the hard question: what happens if the browser dies right here?

● This is the destination, not today. What exists right now is only a React frontend and an Express backend that echoes your message back. The queue, the worker, the agent, the database — none of it is built yet. This walkthrough is the map of where gigachat is going.
