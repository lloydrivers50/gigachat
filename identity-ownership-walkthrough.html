Identity — the message's id. It names the message so the browser can key the bubble, and so anything later (an edit, an assistant reply, an SSE update) can point at the right message.

Ownership — which side mints that id. The browser needs an id the instant the user types (to render now); the server needs one to store a row. The bug is when both mint independently — then one message carries two ids and nothing links them.
